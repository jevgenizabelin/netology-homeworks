Задание № 2.

1. Менеджер по продукту сообщает, что поступило много заказов, связанных с разработкой нового функционала интернет магазина.
2. Принято решение, о необходимости увеличить штат сотрудников на позициях разработчиков и тестировщиков.
3. Разработка плана и графика выпуска новых релизов.
4. DevOps инженер выбирает необходимые инструменты, сервисы и технологии, которые необходимы для успешной разработки и внедрения нового функционала. 
У разработчиков должна быть одинаковая среда разработки, настроены репозитарии Git для dev и prod. 
У тестировщиков одинаковые инструменты для тестирования.
Должна быть настроена система мониторинга и логирования о результатах прохождении кода необходимых тестов.
5. DevOps инженер пишет код инфрастуктуры, для автоматического развертования тестовой платформы в облаке, для проверки нового функционала.
6. Разработчики пишут код нового функционала и выкладываю реультат в dev репоцитарий Git.
7. DevOps инженер настраивает автоматическую доставку кода из dev репозитария Git на тестирование и после успешного тестирования, доставку кода на тестовую платформу.
8. Менеджер по продукту одобряет изменения и принимается решение о дате внедрения изменения на продакшн.
9. DevOps инженер пишет код инфрастуктуры, для автоматического развертования продакшн платформы в облаке.
10. Разработчики отправляют протестированный и одобренный код в prod репозитарий Git.
11. DevOps инженер настраивает автоматическую доставку кода из prod репозитария Git на продакшн платформу.
12. DevOps инженер также настраивает мониторинг, логирование и бэкап продакшн платформы.
13. В случае необходимости или сбоя работы интренет магазина, по причине внедрения нового функционала, настроена возможность откатится на предыдущий релиз.
14. На протяжении всего периода разработки, DevOps инженер контактирует со всеми отделами.
